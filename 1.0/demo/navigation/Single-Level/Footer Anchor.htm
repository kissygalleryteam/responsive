Footer Anchor

这种解决方案把导航列表放在站点的底部，但在页头包含一个指向页尾导航的锚链接。这种方式清除了为核心内容清除出来很多空间同时提供对导航的快速访问。

优点

  • 容易实现- Simple anchor on top. Nav list on the bottom. That’s pretty damn easy.
  • 不依赖js- which means less testing and far better support.
  • 需要扩充的css很少 – Thanks to absolute or fixed positioning, moving the footer nav up to the top for large screens is a piece of cake.
  • 页头只有一个按钮- A simple menu icon or link takes up very little room in the header, which frees up plenty of space for the core content

缺点

  • 锚点跳转可能会让人觉得很怪或是迷惑 – 快速的跳到站点底部可能会让人迷惑.
  • 不优美- this seems weird to say, but other methods like the toggle method have a bit of sexy to them. A jarring jump, while awesomely practical, isn’t the elegant interaction mobile users have gotten used to from interacting with those highly-polished native apps.
